# FRAMEWORKS — 思考フレームワーク体系

## 重要な設計知見: Skillには2つの役割がある

Constraint Matrixに対して、フレームワークは以下のいずれか(または両方)を担う。
各SkillはSKILL.mdの冒頭で自分の役割(role)を宣言しなければならない。

- **generator(行生成器)**: 設計選択肢の行を新しく生み出す・変異させる
- **evaluator(列評価器)**: 担当列のPASS/FAIL/UNKNOWNを判定する

この区別がないと「アイデアは出るが評価されない」か
「評価ばかりで行が増えない」のどちらかに偏る。

## 注記: red-teamは本表に載らない
敵対的レビュー(red-team)はフレームワークではなくコア機構であり、
core/red-team/ に常駐する(AD-06)。本表のSkillが行を生成・評価するのに対し、
red-teamは却下のみを行う第三の役割(gate)である。

## イノベーション成因との対応(選定の妥当性)

Tier 1はもともと構想文書由来であり、後からイノベーション研究の主要成因と
突き合わせて妥当性を検証した(2026-07-18)。結果:

| 成因 | 担い手 | 状態 |
|---|---|---|
| 新結合(シュンペーター) | Swansonリンク探索(Phase 2機構)+形態分析Skill | 実装済 |
| 矛盾の突破(TRIZ) | triz | 実装済 |
| 前提の失効(第一原理) | first-principles | 実装済 |
| 未充足ジョブ(JTBD) | jobs-to-be-done | 実装済 |
| 過剰供給・非消費 | blue-ocean | 実装済 |
| 制約駆動 | frugal-innovation | 実装済 |
| タイミング(Why now) | red-team柱5(守り)+蘇生スキャン(攻め、AUTOPILOT定期処理) | 実装済 |
| 組織・普及 | 対象外(Non-goal: 本キットは発見と審査まで) | 意図的除外 |

したがって「6本+コア機構」で、対象内の主要成因はすべて実装済みとなった。

## 採用の規律

- 新Skillの追加条件: 「どの列を埋めるか」または「どう行を生成するか」を
  1文で宣言できること。できないフレームワークは思想としては立派でも
  本システムでは採用しない(分析ごっこの温床になるため)
- Tier 2の昇格判断: EXTRACTION_PROTOCOL表1で必要性が2回以上観測されたもの
