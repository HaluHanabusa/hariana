# Validation Problems — 答えの分かっている検証問題集

Skillの妥当性検証用フィクスチャ。各Skillの完全実装時、これらの問題に
適用して「実在する既知の解」に到達できるかを確認する。
到達できないSkillは実装不備とみなす(問題側を疑う前にSkillを疑う)。

規約: 問題は常に複数ドメインで用意する。単一ドメインでしか通用しない
Skillは汎用ライブラリに置けない。

## V-01 (光学ドメイン) — 対象Skill: triz
- 矛盾: [改善: 視野角FoV] ⇔ [悪化: 光効率](エタンデュ保存則による)
- 期待される経路: 物理矛盾「全画面が高輝度であれ、かつ低消費電力であれ」
  → 空間による分離 / 原理3(局所的性質)
- 既知の解: フォービエイテッド表示(視線中心のみ高解像度・高輝度)

## V-02 (造船ドメイン) — 対象Skill: triz
- 矛盾: [改善: クレーンブームの到達距離] ⇔ [悪化: 自重による撓みと重量]
- 期待される経路: 原理1(分割) / 原理7(入れ子) / 原理40(複合材料)
- 既知の解: テレスコピックブーム(入れ子式の段階伸縮)、複合材ブーム

## V-03 (ソフトウェアドメイン) — 対象Skill: triz
- 矛盾: [改善: APIの刷新・近代化] ⇔ [悪化: 既存クライアントの互換性]
- 期待される経路: 原理24(仲介) / 時間による分離
- 既知の解: アダプタ層・APIバージョニング(v1を残したままv2を導入)

## V-04 (光学ドメイン) — 対象Skill: first-principles
- 前提の検証: 「ARグラスには両眼ディスプレイが必要」は物理法則か業界慣習か
- 期待される分解: 表示の物理要件(網膜に十分な光子を届ける)まで分解すると
  両眼は慣習であり、単眼・耳掛け型という再構成が選択肢に浮上する
- 既知の解: 単眼型スマートグラス(実在する製品カテゴリ)

## V-05 (小売ドメイン) — 対象Skill: blue-ocean
- 対象: 理髪業界の競争要素(洗髪・予約・マッサージ・接客時間…)
- 期待される経路: ERRC「取り除く」の強制適用
- 既知の解: QBハウス(洗髪等を除去し10分カットに特化)

運用ルール: 新しいSkillを完全実装する際は、最低2ドメイン分の
検証問題をここに追加してから実装に入ること(テストファースト)。

## V-06 (製造・エネルギードメイン) — 対象Skill: first-principles
- 前提の検証: 「EV用バッテリーは高価であり、大きくは下がらない」は原理か慣習か
- 期待される分解: セル価格を材料の商品市況(リチウム・ニッケル・アルミ等)まで
  分解すると、材料原価は完成セル価格の一部にすぎず、残りは構造・製造・流通の
  慣習コストと判明する
- 既知の解: セル形状・工場設計の再構成による大幅なコスト低減(実際に業界で発生)

## V-07 (食品小売ドメイン) — 対象Skill: jobs-to-be-done
- 対象: 朝に売れるミルクシェイクの改善方向
- 期待される経路: 購買状況の観察 → ジョブ「退屈な通勤運転を片手でしのぎ、
  昼まで空腹を防ぎたい」の特定 → 競合は他のシェイクではなくバナナ・ベーグル・
  スニッカーズと判明
- 既知の解: シェイクを「より濃く・長持ちに」する改善(機能競争の逆方向)

## V-08 (工具ドメイン) — 対象Skill: jobs-to-be-done
- 対象: 電動ドリルの顧客価値
- 期待される経路: 「顧客が欲しいのはドリルではなく穴(さらに言えば穴の先の
  棚や達成感)」への言い換え → 機能/感情/社会ジョブの分解
- 既知の解: ジョブ起点の代替(簡易アンカー、施工サービス、仕上がり保証)が
  ドリル性能競争の外に存在する

## V-09 (ゲーム機ドメイン) — 対象Skill: blue-ocean
- 対象: 高性能グラフィック競争が続く家庭用ゲーム機市場
- 期待される経路: 競争要素(描画性能・処理速度・大作タイトル…)の
  戦略キャンバス → ERRCで「描画競争を減らし、体を動かす遊びと家族層
  (それまでゲームをしなかった人)を付け加える」
- 既知の解: 任天堂Wii(性能競争から降りて非顧客層を取った実在の成功例)

## V-10 (医療機器ドメイン) — 対象Skill: frugal-innovation
- 対象: 数百万円級の病院用心電計を、農村の巡回診療で使えるようにしたい
- 期待される経路: 機能の序列化(中核ジョブ=心電図を測って印刷する、だけ残す)
  → 汎用部品への置換(市販のプリンタ部品等) → 携帯型・低価格の再構成
- 既知の解: 携帯型低価格心電計(実在する製品カテゴリ。数十分の一の価格帯)

## V-11 (金融ドメイン) — 対象Skill: frugal-innovation
- 対象: 銀行口座を持てない人が送金できる仕組み
- 期待される経路: 「銀行支店・ATM・スマホが必要」という前提を資源制約で
  そぎ落とす → 手元に既にある資源(通話用の携帯電話と町の売店)だけで再構成
- 既知の解: フィーチャーフォンのSMSによるモバイル送金(実在し普及した方式)

## V-12 (飲料ドメイン) — 対象Skill: morphological-analysis
- 対象: コーヒーの淹れ方の設計空間
- 期待される軸分解: 温度(加熱/非加熱) × 圧力(加圧/常圧) × 方式(浸漬/透過)
- 既知の解: 直積の中に「非加熱×常圧×浸漬」= 水出しコーヒー、
  「加熱×加圧×透過」= エスプレッソが自然に現れること(実在の組合せを
  網羅表が再発見できるかの確認)

## V-13 (移動ドメイン) — 対象Skill: morphological-analysis
- 対象: 一人の短距離移動の設計空間
- 期待される軸分解: 動力(人力/電動) × 姿勢(立つ/座る) × 車輪数(1〜4)
- 既知の解: 直積の中に「電動×立つ×2輪」= 電動キックボードが現れること
