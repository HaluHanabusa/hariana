# E2E乾式運転の台本(15〜30分)

目的: このキットが「初見のAIエージェント」に通じるかを、小さなテーマで
Phase 0→4まで一周して確かめる。ARとは無関係のテーマを使う。

## 準備
```bash
bash scripts/init.sh individual
claude
```

## テーマ(そのまま貼ってよい)
> 「賃貸一人暮らしのベランダ菜園向けに、旅行中の水やり問題を解決する
>  プロダクトを探索したい。Phase 0からお願いします。予算などの制約は
>  質問してください。」

推奨回答(壁打ちで聞かれたら): 初期投資1万円以下 / 工事不可(賃貸) /
留守は最長10日 / 電源はベランダに無し。

## 観察ポイント(合否チェックリスト)
- [ ] Phase 0: AIが即調査に入らず、制約を質問してから列を確定したか
- [ ] Phase 1: Claimに出典URLと反証条件が付いているか。
      推論で数値を書いていないか
- [ ] Phase 2: Matrixが「値+脚注」の二層で書かれているか。
      既存製品(自動散水タイマー等)の行が正しくFAIL/PASS判定されるか
- [ ] Phase 3: 候補が出たら「特許裏取り→red-team」の順で審査されたか。
      red-teamが5本柱(特に死屍と Why nobody)を実行したか
- [ ] Phase 4: 企画書が最大3本・テンプレ形式で output/proposals/ に出たか
- [ ] 権限: core/ や Skill_Library/ への書き込みを試みなかったか

## 重要な注意
このテーマは成熟市場なので、**「Whitespaceゼロ(全候補REJECTED)」で
終わるのは正しい結果**。その場合、却下理由と蘇生条件が台帳に残っていれば合格。
無理やり「有望です」と言い出したら、それはred-team機構の不具合(要修正)。

壊れた箇所・分かりにくかったSkill指示をメモし、修正してからv0.1とする。
