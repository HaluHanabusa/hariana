# 形態分析(モルフォロジカル分析)

言葉の意味: 対象を「設計軸(どんな性質で決まるかの軸)」に分け、
軸ごとの選択肢を掛け合わせて、あり得る組み合わせを表で全部並べる方法。
「思いついた案」ではなく「あり得る案の全体」から探すための道具。

## このSkillが呼ばれる場面
- Matrixの行が少ない・既存製品の写しばかりで、盤面が痩せているとき
- 「Whitespaceゼロ」と結論する前に、探索空間を敷き詰めたと言い切りたいとき
- Swansonリンク(GLOSSARY参照)で見つけた要素を、組み合わせとして展開したいとき

## 手順

### Step 1: 機能を動詞で書く
対象を「何をするものか」の動詞で書き直す(例: コーヒーを淹れる →
「豆の成分を水に移す」)。製品名のままだと既存の形に引きずられる。

### Step 2: 設計軸を3〜6本選ぶ
その機能を左右するパラメータを軸にする(例: 温度・圧力・方式)。
軸は互いに独立に選べるものにする(片方を決めると他方が決まる軸は1本に統合)。

### Step 3: 各軸の値を列挙する
いま市場にある値だけでなく、**物理的に可能な値をすべて**挙げる。
値の根拠(実在例か、物理的に可能な理由)をClaim IDか1文で添える。

### Step 4: 直積を展開する
全組み合わせを表にする。組み合わせ数が thresholds.morph_max_combinations
(既定60)を超える場合は、軸を分割して2段階で回すか、値を絞る。

### Step 5: ふるいにかける(捨てた理由も記録する)
- 物理矛盾の組み合わせ → 除外(理由を1文で記録。黙って消さない)
- 既存製品が占める組み合わせ → 「占有」マークを付ける
- 残った空きマス → Matrixの行として登録する

## 出力契約
- 生成行には `morph_axes: {軸: 値, ...}` を付与する
- 軸の定義・値の根拠・除外理由の記録が無い展開は無効とする
- 「空きマス」はまだWhitespaceではない。列の評価(他Skill)と針通し
  (特許裏取り→red-team)を経て初めて候補になる
